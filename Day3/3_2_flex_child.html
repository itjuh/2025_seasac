flex-direction property

order

flex자식의 크기를 지정하는 속성
width와 유사하나 우선순위가 높음
auto는 hug와 같음
flex-grow : 전체값에 남은부분의 비율로 나눠가지는 속성
0* / number
커질래? 그래서 0이면 안늘어나는구나
원래크기+남은값을 grow값 비율로 나눠서 가짐

600px의 부모에 100px box를 2:1:1 flex-grow하는 경우
box:nth-child(1) 100+(300/4)*2=250px
box:nth-child(2)+nth-child(3) 100+(300/4)*1=175px
flex-shrink : 전체가 넘친 값에서 비율을 버리는 속성
1* / number
작아질래?
원래크기-넘친값을 shrink값 비율로 나누어서 뺌

600px의 부모에 400px의 박스를 3:1:1:로 flex-shrink하는 경우
1: 400 - (300/5)*3 = 40px
2,3: 400 - (300/5)*1 = 280px
flex : 축약형
auto (0 1 auto)
flex-grow / flex-shrink / flex-basis
flex : none 0 0 auto
flex : initial 0 1 auto
flex : auto 1 1 auto
